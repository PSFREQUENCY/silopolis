### OKB Accumulation Protocol

The vault's **primary rule**: OKB balance must never drain.

- **Floor** `0.00222 OKB` — trading suspended below this threshold
- **Buffer** `0.00666 OKB` — buyback mode activates when below (3× floor)
- **Buyback** — only TRADER-1 and SUSTAINER-8 execute; others observe and forecast
- **Sizing** — 40% of available USDT in aggressive campaign mode (not fragmented across 9 agents)
- **Profit capture** — 50% of every winning trade stays in vault, 50% compounds

## x402 Micropayments

Agents acquire skill relics via **HTTP 402 micropayment protocol** — agent-to-agent payments, fully on-chain:

```
1. Agent needs "Oracle Lens" relic
2. Issues GET /api/skill/oracle-lens
3. Server responds: HTTP 402 + WWW-Authenticate: x402 ...
4. SILO-SKILL-3 intercepts the 402 challenge
5. Swaps available token → required payment token via OnchainOS
6. Replays request with X-Payment: <signed-receipt> header
7. Skill relic stored on-chain in SkillMarket.sol
8. Agent reputation updated: collaboration +1, proficiency unlock
```

This creates a live **agent micropayment economy** — agents earn OKB from trades, spend it to acquire skills, use those skills to trade better. x402 transactions glow purple in the Living Brain.

## Self-Healing & Safety

- **5 consecutive losses** → 8-hour pause (circuit breaker in `core/risk.py`)
- **Threat score ≥ 76** → Gemini decision blocked by GUARD before execution
- **Daily OKB budget** → hard cap per tier, resets every 24h
- **OKB floor** → trading hard-stops below `0.00222 OKB`
- **Buyback protocol** → when balance below buffer, buying OKB overrides all else
- **50% profit capture** → every win: 50% stays in vault, 50% compounds
- **Immutable audit trail** → every decision logged to SQLite with timestamp, agent, outcome

## Setup

Requires: Python 3.12+, Node.js 18+, Foundry (for contracts)

```bash
# 1. Configure environment
cp .env.example .env
# Required: GEMINI_API_KEY, ONCHAINOS_API_KEY, AGENT_WALLET_ADDRESS, XLAYER_RPC_URL

# 2. Install dependencies

# 3. Run a single heartbeat cycle
python3 -m core.heartbeat

# 4. Dashboard dev server
```
