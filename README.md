### OKB Accumulation Protocol

The vault's **primary rule**: OKB balance must never drain.

- **Floor** `0.00222 OKB` — trading suspended below this threshold
- **Buffer** `0.00666 OKB` — buyback mode activates when below (3× floor)
- **Buyback** — only TRADER-1 and SUSTAINER-8 execute; others observe and forecast
- **Sizing** — 40% of available USDT in aggressive campaign mode (not fragmented across 9 agents)
- **Profit capture** — 50% of every winning trade stays in vault, 50% compounds

## Self-Healing & Safety

- **5 consecutive losses** → 8-hour pause (circuit breaker in `core/risk.py`)
- **Threat score ≥ 76** → Gemini decision blocked by GUARD before execution
- **Daily OKB budget** → hard cap per tier, resets every 24h
- **OKB floor** → trading hard-stops below `0.00222 OKB`
- **Buyback protocol** → when balance below buffer, buying OKB overrides all else
